# Jarvis 메인 프로젝트 교체 디자인

## 목표

기존 랜딩 페이지의 시각 언어와 카드 구조를 유지하면서 두 번째 메인 프로젝트인 원천징수 레퍼런스를 Jarvis로 교체한다.

## 구성

- 메인 프로젝트 1은 `EHR Harness`를 유지한다.
- 메인 프로젝트 2는 `Jarvis`로 교체한다.
- 두 프로젝트는 “AI가 레거시 HR을 이해하게 만든 기반 → 그 기반으로 만든 실제 사내 지식 플랫폼”의 연속된 이야기로 제시한다.
- Jarvis CTA는 `https://jarvis.minseok91.cloud/`로 연결한다.
- 라이브 서비스가 아직 준비되지 않아도 프로젝트를 이해할 수 있도록 제공된 실제 Jarvis 대시보드 스크린샷을 카드 이미지로 사용한다.

## 변경 범위

- `src/data/siteContent.ts`: 원천징수 프로젝트 객체를 Jarvis 프로젝트 객체로 교체한다.
- `public/images/jarvis-thumb.png`: 제공된 Jarvis 스크린샷을 배포 자산으로 추가한다.
- 기존 컴포넌트, 레이아웃, 애니메이션 및 디자인 토큰은 변경하지 않는다.

## 문구 방향

Jarvis를 단순 챗봇이 아니라 사내 업무 시스템과 Git 기반 지식 위키를 결합한 엔터프라이즈 지식 플랫폼으로 소개한다. 문제는 흩어진 업무 지식과 출처 없는 AI 답변, 접근 방식은 근거 문서를 직접 탐색하는 tool-use agent와 Git 이력, 결과는 검색·인용·검토 가능한 단일 업무 지식 환경으로 정리한다.

## 검증

- `npm run lint`
- `npm run build`
- Git diff에서 원천징수 카드가 완전히 제거되고 Jarvis URL 및 이미지 경로가 반영됐는지 확인한다.
