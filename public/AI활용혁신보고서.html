하네스 엔지니어링

AI를 운영 가능하게 만드는 기술

프롬프트 vs 컨텍스트 vs 하네스

프롬프트 엔지니어링

→ "무엇을 지시할 것인가" (입력 최적화 - 대화창에서 지시문을 잘 입력하기)

컨텍스트 엔지니어링

→ "무엇을 보게 할 것인가" (정보 큐레이션 - 필요한 모든 정보를 주기)

하네스 엔지니어링

→ "어떤 규칙·도구·검증 안에서 일하게 할 것인가"
   운영 체계 설계

하네스의 5가지 구성 요소

① 정보 세팅
참고 문서·업무 맥락 제공
② 도구 세팅
승인된 시스템만 사용 허용
③ 절차 세팅
업무 순서·매뉴얼 사전 정의
④ 검증 세팅
테스트·로그·승인 게이트
⑤ 개선 세팅
반복 사용 → 규칙 보완
"속도는 AI가 만들고, 신뢰는 하네스가 만든다"

사람의 역할: 결과물을 직접 고치는 'in the loop'에서 → 하네스를 바꾸는 'on the loop'로 이동

EHR5 하네스 아키텍처

운영 중인 대형 HR 레거시 시스템에서 AI가 바로 구현하지 않고, 화면-DB-프로시저-품질 게이트를 먼저 통과하게 만드는 문서형 control plane

개요

EHR5는 운영 중인 Java/JSP/MyBatis/Oracle 기반 HR 시스템으로, AI 활용의 핵심은 생성 속도보다 레거시 영향 범위 통제에 있다.

그래서 요청 분류, 화면-DB 추적, 프로시저 리스크 판별, 공통 서비스 경계, 수동 QA와 품질 게이트를 하나의 하네스로 엮었다.

AI 사용량 관리 체계

도입의 마지막 1마일 — AI 도구를 사줬으면, 쓰게 만들어야 한다

운영 방안

1

계정 부여

팀별 Claude Max ($200/월) 계정 부여

2

사용 목표 설정

주간 70% 이상 사용 (목표 100%)
팀장이 주기적으로 사용량 확인

3

자동 보고 체계

매일 오전 9시 스케줄링
사용량 파악 → 목표 대비 달성률 계산
파트장에게 자동 메일 발송

4

사용량 미달 시

파트장 → 팀원에게 활용 독려
활용 사례 공유 + 교육 지원

왜 '강제'가 필요한가

  • •도구를 사 줘도 안 쓰면 비용 낭비
  • •초기 습관 형성이 가장 중요 (3주 이상 매일 사용 → 자발적 전환)
  • •LG CNS도 '조직 단위 의무 사용'으로 성과 달성
  • •개인 자율에 맡기면 격차만 벌어짐

정착의 핵심

초기 3주 매일 사용 습관 형성
관리 기준 주간 70% 이상 사용

이 자동 보고 시스템 자체도 Claude가 만들 수 있음

→ '사용량 확인해서 파트장들에게 메일 보내라'라고 지시하면 스케줄링 로직 + 메일 발송 코드까지 자동 생성

AI 도입의 첫 시작은 '쓰게 만드는 것'
